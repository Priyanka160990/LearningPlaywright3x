# Source Code vs Byte Code vs Binary Code

## Example Reference

**File:** `01_Chapter_Basics/01_HelloWorld.js`

```javascript
console.log("Hello World : The Testing Academy");
```

---

## Comparison Table

| Aspect | Source Code | Byte Code | Binary Code (Machine Code) |
|--------|-------------|-----------|----------------------------|
| **Definition** | Human-readable code written by a programmer in a high-level language | Intermediate representation between source code and machine code | Instructions directly executed by the CPU in binary (0s and 1s) |
| **Readability** | Readable by humans | Difficult to read, but some tools can decompile it | Not human-readable at all |
| **Example** | `console.log("Hello World : The Testing Academy");` | `ldc "Hello World : The Testing Academy"`<br>`invokevirtual console.log`<br>`return` | `10101000 00000001`<br>`10101000 00000010`<br>`11000011` |
| **Who writes it** | Programmers / Developers | Generated by a **compiler** (e.g., V8, Javac, GCC) | Generated by an **assembler** or **bytecode interpreter** |
| **Language level** | High-level (JavaScript, C, Java, Python) | Intermediate-level | Low-level |
| **Execution** | Cannot be executed directly | Needs a **Virtual Machine** or **interpreter** (JVM, V8 engine, .NET CLR) | Directly executed by the **CPU** |
| **Portability** | Portable across platforms (write once, run anywhere where runtime exists) | Portable — works on any platform that has the corresponding VM | Not portable — specific to CPU architecture (x86, ARM, etc.) |
| **Speed** | — (not executed directly) | Faster than interpretation alone, slower than binary | **Fastest** — native CPU execution |
| **Extension** | `.js`, `.py`, `.java`, `.c`, `.cpp` | `.class` (Java), `.pyc` (Python), `.wasm` (WebAssembly) | `.exe`, `.dll`, `.so`, `.out`, `.bin` |
| **Abstraction level** | Highest abstraction | Medium abstraction | Lowest abstraction (hardware level) |

## How They Relate (Flow Diagram)

```
Source Code (.js)
      |
      ▼
  [Parser / Compiler — V8 Engine]
      |
      ▼
  Byte Code (Ignition Interpreter)
      |
      ▼
  [JIT Compiler — TurboFan]
      |
      ▼
  Binary / Machine Code (CPU executes)
```

### Step-by-step for `Hello World : The Testing Academy`:

1. **Source Code** — You write `console.log("Hello World : The Testing Academy");` in a `.js` file.
2. **Byte Code** — The V8 JavaScript engine parses your source and compiles it into intermediate byte code. This is platform-independent and compact.
3. **Binary Code** — The JIT (Just-In-Time) compiler (TurboFan in V8) converts hot (frequently used) byte code into native machine instructions (0s and 1s) that your specific CPU understands.

## Key Takeaway

| Concept | One-line summary |
|---------|------------------|
| **Source Code** | What you write |
| **Byte Code** | What the VM understands |
| **Binary Code** | What the CPU executes |
